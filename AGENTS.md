# BowlingWeb - AI エージェント向けガイド

**焦点：可読性の高いコード品質の維持と向上**

## プロジェクト概要

- **種類**: Flask + Vanilla JavaScript + HTML/CSS
- **用途**: ボウリングチャレンジゲーム（お題付きスコアリングシステム）
- **レーン数**: 最大 4 レーン
- **言語**: Python (バックエンド)、JavaScript (フロントエンド)、日本語ドキュメント

## 読みやすさの原則

このプロジェクトでは、以下の原則に従ってコードを記述してください：

### 1. セクション分割（必須）

**ルール**: 機能やモジュールは明確なセクションヘッダーで分割する

```python
# Python の場合
# ==================================
# セクション名
# ==================================

// JavaScript の場合
// =====================================
// セクション名
// =====================================
```

**なぜ**: 200行以上のファイルでも、セクションヘッダーで全体構造が一目で分かり、ジャンプが容易になります。

### 2. 関数・変数の命名

**ルール**: 日本語でも英語でも、**意図が明確** な名前を付ける

❌ 避けるべき例:
```python
def calc(x, y):  # 何を計算？
    pass

r = fetch_data()  # r は何？
```

✅ 推奨例:
```python
def calculate_bonus_score(base_score, difficulty_multiplier):
    pass

response = fetch_api_config()  # 日本語なら response_config でもOK
```

### 3. コメント戦略

**ルール**: 
- **"何をしているか"** よりも **"なぜそうするのか"** をコメントに書く
- セクションの目的は日本語で1行記述
- 複雑ロジックのみ詳説

例:
```python
def load_config(filename=CONFIG_FILE):
    """
    odai.txt を読み込む
    形式：
    両目隠し,4.5
    ピン数減少,4
    """
    global OdaiList, ValueList
    
    OdaiList.clear()  # 前回の設定をリセット（重複防止）
    ValueList.clear()
```

### 4. コード フォーマット

**ルール**:
- **インデント**: 空白 4 文字（Python）、2 または 4 文字（JavaScript）
- **行長**: 100 文字未満が目安（長い場合は分割）
- **空行**: セクション間に1行、関数間に1行

### 5. ファイル構成

**Python（server.py）**:
- サーバー設定 → お題データ → ルート定義 → API エンドポイント → ユーティリティ

**JavaScript（client.js など）**:
- グローバル変数 → 設定取得 → 得点送信 → UI イベント → ヘルパー関数

## 具体的な改善ガイド

### 機能追加時
1. 既存のセクションに追加する場合 → 同じセクションヘッダー内で記述
2. 新しい概念 → セクションヘッダーを追加

### 変更時の注意点
- セクションヘッダーを削除・変更しない
- グローバル変数は `teams` のように辞書で管理（配列の羅列は避ける）
- エラーメッセージは日本語で明確に

### リファクタリング時
- 関数の責務を 1 つに（単一責任原則）
- 関数が 30 行以上 → 分割を検討
- ネストが 3 層以上 → 早期 return やヘルパー関数を使用

## API エンドポイント設計

新しい API を追加する際：

```python
@app.route("/api/endpoint")
def endpoint_name():
    """
    エンドポイント説明（何を返すか）
    """
    # 処理
    return jsonify({"status": "success", "data": ...})
```

## テンプレート規約

HTML テンプレートでは：
- `<div class="card">` で論理的な単位をグループ化
- ID は JavaScript 操作用のみ（クエリセレクタで `getElementById` を使用）
- クラスはスタイリング用

## 改善提案の優先度

1. **高**: セクション構造が崩れている、変数名が曖昧
2. **中**: コメント不足、行が長い、関数が大きい
3. **低**: 細かいスタイル調整、型ヒント追加

## 推奨ツール・拡張機能

- **Pylint** または **Black**: Python コード フォーマッタ
- **Prettier**: JavaScript/HTML/CSS フォーマッタ
- **ESLint**: JavaScript 規約チェック（日本語コメントを許容する設定）
